# Product

## Register

product

## Platform

web

## Users

Solo developers and technical writers who need to share ideas visually — code snippets, diffs, terminal sessions, API shapes, database schemas. They're in the middle of building or explaining something and want the output to look intentional, not like a raw screenshot. Primary: developers posting to social media or sharing with teammates. Secondary: technical writers creating documentation visuals.

## Product Purpose

MyCodeSnap is a canvas for composing and exporting beautiful technical visuals. It turns raw developer artifacts — code, diffs, terminals, APIs, database schemas — into polished, shareable images. Success means a developer can go from idea to exported visual in under a minute, without touching a design tool.

## Positioning

The fastest way for developers to make technical content look intentional.

## Brand Personality

Easy, elegant, efficient. The tool should feel like it was made by a developer who cares about craft — opinionated defaults, no friction, no noise. The UI disappears; the content is the hero.

## Anti-references

Overly complex IDEs or design tools (Figma-like panels everywhere). GitHub Gist's bare, unstyled output. Tools that feel like they require a design background to use. Cluttered toolbars that compete with the canvas.

## Design Principles

1. **Content is the hero** — the canvas and its nodes are the product; the chrome serves them.
2. **Opinionated defaults, optional depth** — sensible presets get users to a great result fast; settings panels exist but don't demand attention.
3. **Craft without complexity** — every detail should feel considered, not over-engineered.
4. **Efficiency over ceremony** — no confirmation dialogs for reversible actions, no multi-step flows for simple tasks.

## Accessibility & Inclusion

WCAG AA. Keyboard-navigable canvas controls. Sufficient contrast on all text-bearing UI chrome.
